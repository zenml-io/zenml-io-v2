In the ever-evolving landscape of machine learning operations (MLOps), efficient pipeline management is crucial for successful ML projects. Today, we're excited to introduce a powerful integration that combines the flexibility of ZenML with the robustness of Amazon SageMaker Pipelines. This collaboration enables you to streamline your ML workflows by running ZenML pipelines as Amazon SageMaker Pipelines, a serverless ML orchestrator from AWS.

## Unleashing the Power of SageMaker Pipelines for ZenML

By leveraging this integration, you can take advantage of SageMaker's scalability, robustness, and built-in features to manage your ML pipelines efficiently in production environments. Let's explore the key features and benefits of this integration:

### ZenML Integration Features

<ol id=""><li id=""><strong id="">Seamless Execution</strong>: Run your ZenML pipelines as SageMaker Pipelines with minimal configuration changes.</li><li id=""><strong id="">Effortless Scaling</strong>: Leverage SageMaker's serverless infrastructure to scale your pipeline execution effortlessly.</li><li id=""><strong id="">Enhanced Monitoring</strong>: Track and monitor your pipeline runs using SageMaker's intuitive user interface.</li><li id=""><strong id="">Resource Customization</strong>: Easily customize instance types and resources for your entire pipeline or individual steps.</li><li id=""><strong id="">Stack Component Integration</strong>: Seamlessly use other AWS stack components like S3 for storage and ECR for container management.</li></ol>

### SageMaker Pipelines Advantages

<ol id=""><li id=""><strong id="">Serverless Orchestration</strong>: Enjoy the benefits of serverless, scalable orchestration for your ML workflows.</li><li id=""><strong id="">Built-in Capabilities</strong>: Utilize SageMaker's built-in data processing and model training capabilities.</li><li id=""><strong id="">Visual Pipeline Management</strong>: Monitor and observe your pipelines through a comprehensive visual interface.</li><li id=""><strong id="">AWS Ecosystem Integration</strong>: Seamlessly integrate with other AWS services for end-to-end ML solutions.</li></ol>

## Getting Started

Let's walk through the process of setting up and using the ZenML SageMaker Pipelines integration:

### Step 1: Register a New SageMaker Orchestrator

First, you'll need to register a new SageMaker orchestrator with ZenML:

<div data-rt-embed-type="true"><pre><code fs-codehighlight-element="code" class="language-shell">zenml orchestrator register &lt;ORCHESTRATOR_NAME&gt; \
 &nbsp; &nbsp;--flavor=sagemaker \
 &nbsp; &nbsp;--execution_role=&lt;YOUR_IAM_ROLE_ARN&gt;</code></pre></div>

### Step 2: Authenticate the SageMaker Orchestrator

You have three options for authentication:

<ul id=""><li id=""><strong id="">Service Connector (Recommended)</strong>:</li></ul>

<div data-rt-embed-type="true"><pre><code fs-codehighlight-element="code" class="language-shell">zenml orchestrator connect &lt;ORCHESTRATOR_NAME&gt; --connector &lt;CONNECTOR_NAME&gt;
</code></pre></div>

<ul><li><strong id="">Explicit Authentication</strong>:</li></ul>

<div data-rt-embed-type="true"><pre><code fs-codehighlight-element="code" class="language-shell">zenml orchestrator register &lt;ORCHESTRATOR_NAME&gt; \
 &nbsp; &nbsp;--flavor=sagemaker \
 &nbsp; &nbsp;--execution_role=&lt;YOUR_IAM_ROLE_ARN&gt; \
 &nbsp; &nbsp;--aws_access_key_id=...
 &nbsp; &nbsp;--aws_secret_access_key=...
 &nbsp; &nbsp;--region=...</code></pre></div>

<ul><li><strong id="">Implicit Authentication</strong>: No additional steps needed; auth settings will be used from the running environment implicitly.</li></ul>

### Step 3: Update Your Stack

Update your ZenML stack to use the new SageMaker orchestrator:

<div data-rt-embed-type="true"><pre><code fs-codehighlight-element="code" class="language-shell">zenml stack update -o &lt;ORCHESTRATOR_NAME</code></pre></div>

## Implementing a SageMaker Pipeline

Now, let's look at how to implement a ZenML pipeline that runs on SageMaker:

<div data-rt-embed-type="true"><pre><code fs-codehighlight-element="code" class="language-python">from zenml import step, pipeline
from zenml.integrations.aws.flavors.sagemaker_orchestrator_flavor import (
 &nbsp; &nbsp;SagemakerOrchestratorSettings,
)

@step
def preprocess_data() -&gt; int:
 &nbsp; &nbsp;return 1

@step
def train_model(data: int) -&gt; str:
 &nbsp; &nbsp;return str(data)

@pipeline(
 &nbsp; &nbsp;settings={
 &nbsp; &nbsp; &nbsp; &nbsp;"orchestrator.sagemaker": SagemakerOrchestratorSettings(
 &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;instance_type="ml.m5.large",
 &nbsp; &nbsp; &nbsp; &nbsp; &nbsp; &nbsp;volume_size_in_gb=30,
 &nbsp; &nbsp; &nbsp; &nbsp;),
 &nbsp; &nbsp;}
)
def ml_pipeline():
 &nbsp; &nbsp;input_data = preprocess_data()
 &nbsp; &nbsp;train_model(input_data)

if __name__ == "__main__":
 &nbsp; &nbsp;ml_pipeline()
</code></pre></div>

This example demonstrates how to configure ZenML steps to run on SageMaker Pipelines. It showcases customizing instance types and resources for the entire pipeline. The `preprocess_data` step and `train_model` step will both run on the specified `ml.m5.large` instance with a 30GB volume.
