As machine learning engineers, we're constantly seeking ways to streamline our workflows, improve collaboration, and gain deeper insights into our models. In this blog post, we'll explore how integrating ZenML, an open-source MLOps framework, with [Comet](https://www.comet.com/site/products/ml-experiment-tracking/), a powerful experiment tracking platform, can greatly enhance your ML pipeline management and experimentation process.

## The MLOps Challenge

In the world of machine learning, keeping track of experiments, managing model versions, and maintaining reproducibility can quickly become overwhelming. As your projects scale and your team grows, these challenges only intensify. This is where the combination of ZenML and Comet comes in, offering a robust solution to elevate your MLOps game.

## Enter ZenML and Comet

[ZenML](https://www.zenml.io/) is an extensible, open-source MLOps framework designed to create portable, production-ready machine learning pipelines. It offers a simple, intuitive API that allows you to define your ML workflows as a series of steps, making it easy to manage complex pipelines.

[Comet](https://www.comet.ml/) is a platform for tracking, comparing, explaining, and optimizing experiments and models. It provides a centralized place to log hyperparameters, metrics, code, and artifacts, making it easier to collaborate and iterate on your ML projects.

When combined, these tools offer a powerful solution for managing your entire ML lifecycle, from experimentation to production.

## Why Integrate ZenML and Comet?

<ol id=""><li id=""><strong id="">Seamless Experiment Tracking</strong>: Automatically log metrics, parameters, and artifacts from your ZenML pipelines to Comet experiments.</li><li id=""><strong id="">Enhanced Visibility</strong>: Visualize your pipeline results in Comet's intuitive UI, making it easy to compare runs and share insights with your team.</li><li id=""><strong id="">Simplified Workflow</strong>: Enable Comet tracking in your ZenML steps with minimal code changes.</li><li id=""><strong id="">Improved Collaboration</strong>: Organize experiments with automatic tags based on pipeline names and runs, making it easier for team members to find and understand relevant experiments.</li><li id=""><strong id="">Flexible Configuration</strong>: Fine-tune your experiment tracking setup using ZenML's <a href="https://sdkdocs.zenml.io/0.66.0/integration_code_docs/integrations-comet/#zenml.integrations.comet.flavors.comet_experiment_tracker_flavor.CometExperimentTrackerSettings" id=""><code id="">CometExperimentTrackerSettings</code></a>.</li></ol>

## Getting Started

Let's walk through the process of integrating ZenML and Comet into your ML workflow.

### Step 1: Installation and Setup

First, make sure you have ZenML installed and the Comet integration enabled:

<div data-rt-embed-type="true"><pre><code fs-codehighlight-element="code">pip install zenml
zenml integration install comet -y</code></pre></div>

Next, you'll need to configure your Comet credentials. The recommended way is to use ZenML secrets:

<div data-rt-embed-type="true"><pre><code fs-codehighlight-element="code" class="language-shell">zenml secret create comet_credentials \\
 &nbsp; &nbsp;--workspace=&lt;WORKSPACE&gt; \\
 &nbsp; &nbsp;--project_name=&lt;PROJECT_NAME&gt; \\
 &nbsp; &nbsp;--api_key=&lt;API_KEY&gt;</code></pre></div>

Now, register the Comet experiment tracker in your ZenML stack:

<div data-rt-embed-type="true"><pre><code fs-codehighlight-element="code" class="language-shell">zenml experiment-tracker register comet_tracker \\
 &nbsp; &nbsp;--flavor=comet \\
 &nbsp; &nbsp;--workspace={{comet_credentials.workspace}} \\
 &nbsp; &nbsp;--project_name={{comet_credentials.project_name}} \\
 &nbsp; &nbsp;--api_key={{comet_credentials.api_key}}</code></pre></div>

After registering the Comet experiment tracker, you need to create a new stack that includes this component. Here's how you can do that with a default orchestrator and artifact store:

<div data-rt-embed-type="true"><pre><code fs-codehighlight-element="code" class="language-shell">zenml stack register custom_stack -e comet_tracker -o default -a artifact --set
</code></pre></div>

The `--set` flag at the end sets this new stack as your active stack.

### Step 2: Integrating Comet in Your ZenML Pipeline

Here's an example of how to use the Comet experiment tracker in a ZenML pipeline:

<div data-rt-embed-type="true"><pre><code fs-codehighlight-element="code" class="language-python">from comet_ml.integration.sklearn import log_model
from sklearn.model_selection import train_test_split
from sklearn.datasets import load_iris
from sklearn.svm import SVC
from sklearn.metrics import accuracy_score
from zenml import pipeline, step
from zenml.client import Client
from zenml.integrations.comet.experiment_trackers import CometExperimentTracker

# Get the experiment tracker from the active stack
experiment_tracker: CometExperimentTracker = Client().active_stack.experiment_tracker


@step(experiment_tracker=experiment_tracker.name)
def train_model() -&gt; SVC:
 &nbsp; &nbsp;iris = load_iris()
 &nbsp; &nbsp;X_train, _, y_train, _ = train_test_split(
 &nbsp; &nbsp; &nbsp; &nbsp;iris.data, iris.target, test_size=0.2, random_state=42
 &nbsp; &nbsp;)
 &nbsp; &nbsp;model = SVC(kernel="rbf", C=1.0)
 &nbsp; &nbsp;model.fit(X_train, y_train)

 &nbsp; &nbsp;# Log the model to Comet
 &nbsp; &nbsp;log_model(
 &nbsp; &nbsp; &nbsp; &nbsp;experiment=experiment_tracker.experiment,
 &nbsp; &nbsp; &nbsp; &nbsp;model_name="SVC",
 &nbsp; &nbsp; &nbsp; &nbsp;model=model,
 &nbsp; &nbsp;)

 &nbsp; &nbsp;return model


@step(experiment_tracker=experiment_tracker.name)
def evaluate_model(model: SVC):
 &nbsp; &nbsp;iris = load_iris()
 &nbsp; &nbsp;_, X_test, _, y_test = train_test_split(
 &nbsp; &nbsp; &nbsp; &nbsp;iris.data, iris.target, test_size=0.2, random_state=42
 &nbsp; &nbsp;)
 &nbsp; &nbsp;y_pred = model.predict(X_test)
 &nbsp; &nbsp;accuracy = accuracy_score(y_test, y_pred)

 &nbsp; &nbsp;# Log metrics using Comet
 &nbsp; &nbsp;experiment_tracker.log_metrics({"accuracy": accuracy})

 &nbsp; &nbsp;return accuracy


@pipeline
def ml_pipeline():
 &nbsp; &nbsp;model = train_model()
 &nbsp; &nbsp;accuracy = evaluate_model(model)


if __name__ == "__main__":
 &nbsp; &nbsp;ml_pipeline()</code></pre></div>

In this example, we've created two steps: `train_model` and `evaluate_model`. Both steps are decorated with `@step(experiment_tracker=experiment_tracker.name)`, which enables Comet tracking for these steps.

### Step 3: Customizing Experiment Settings

Now, when you run your pipeline, ZenML will automatically create Comet experiments for each step where tracking is enabled. You can view these experiments in the Comet UI by visiting `https://www.comet.com/{WORKSPACE_NAME}/{PROJECT_NAME}/experiments/`. For more fine-grained control over your Comet experiments, you can use `CometExperimentTrackerSettings`:

<div data-rt-embed-type="true"><pre><code fs-codehighlight-element="code" class="language-python">from zenml.integrations.comet.flavors.comet_experiment_tracker_flavor import (
 &nbsp; &nbsp;CometExperimentTrackerSettings,
)

comet_settings = CometExperimentTrackerSettings(
 &nbsp; &nbsp;tags=["production", "v1.0"],
 &nbsp; &nbsp;run_name="my_custom_run_name",
)

@step(
 &nbsp; &nbsp;experiment_tracker="comet_tracker",
 &nbsp; &nbsp;settings={
 &nbsp; &nbsp; &nbsp; &nbsp;"experiment_tracker": comet_settings
 &nbsp; &nbsp;}
)
def my_custom_step():
 &nbsp; &nbsp;# Your step logic here
 &nbsp; &nbsp;...</code></pre></div>

This allows you to add custom tags, set specific run names, and configure other [Comet-specific settings for your experiments.](https://docs.zenml.io/stack-components/experiment-trackers/comet)
