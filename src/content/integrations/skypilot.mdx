# easily configure your runtime
skypilot_settings = {
 &nbsp; &nbsp;cpus="2",
 &nbsp; &nbsp;memory="16",
 &nbsp; &nbsp;accelerators="V100:2",
 &nbsp; &nbsp;...
}


@pipeline(
 &nbsp; &nbsp;settings={
 &nbsp; &nbsp; &nbsp; &nbsp;"orchestrator.vm_aws": skypilot_settings,
 &nbsp; &nbsp; &nbsp; &nbsp;# "orchestrator.vm_gcp": skypilot_settings,
 &nbsp; &nbsp; &nbsp; &nbsp;# "orchestrator.vm_azure": skypilot_settings,
 &nbsp; &nbsp; &nbsp; &nbsp;# "orchestrator.vm_lambda": skypilot_settings
 &nbsp; &nbsp;}
)
</code></pre></div>This code snippet demonstrates how to configure the SkyPilot VM orchestrator for a ZenML pipeline. The Skypilot settings allows you to specify the desired resources (CPUs, memory, accelerators) and other parameters like spot instances, region, disk size, and auto-stop behavior. These settings can be applied at the pipeline level or individual step level for fine-grained resource allocation.
